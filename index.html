Google Sheets 串接方式

正式上線時，最穩的做法是把 Google Sheets 當資料來源。你維護表格，網站負責讀取並顯示。這樣比一開始做完整後台更省成本，也更符合持續更新的需求。

步驟流程

  1. 建立一份 Google Sheets，第一列固定為欄位名稱。
  2. 欄位建議包含：name、area、category、price、signature、tags、rating、favorite、note。
  3. 將試算表整理為乾淨資料表，不要合併儲存格。
  4. 用 Google Apps Script 或公開 JSON 方式，把資料輸出給前端讀取。
  5. 網站前端用 JavaScript 抓資料後渲染卡片、篩選器與統計數字。

你最適合的版本

推薦 Google Sheets + Apps Script Web App：最平衡，維護容易，也可保留日後擴充性。

如果只是個人使用，可先用公開 JSON 或手動匯出 CSV。

如果未來要多人協作、權限管理、圖片與地圖功能，再升級到 Supabase 或 Firebase。
